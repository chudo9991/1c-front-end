7.1 Browserify

7.1.1 Использование пакетов

Клиентской части крупных проектов используются десятки библитек и модулей JavaScript, зависящих друг от друга. Чтобы оперделить порядок их подключения вручную, требуется проделать немалую работу. Кроме того, следить чтобы всё глобальные переменные имели уникальные имена, при таком большом числе файлов неудобно.

Для решения этих проблем в Node.js применяется концепция пакетов, подключаемых функцией require. Поскольку в браузере аналогичной системы нет, вместо ней используется сборщик Browserify. Сейчас мы будем его устанавливать.

Это делаем в Windows Power Shell и Node.js!

  1. Запустить Windows PowerShell
  2. Набрать 'node --version'
  3. Набрать 'npm install --global browserify'.
  4. Перейти в папку "example". Использовать команды cd (сменить директорию) и dir (список файлов в директории).
  5. В папке "заготовки" набрать 'npm init --force'.Эта папка станет нашим сервером.
  6. Установить JQuery набрав 'npm install jquery --save'.

Теперь создадим страницу с кнопкой.

        Листинг 7.1 index.html 

        <!DOCTYPE html>
        <html>
        <head>
            <meta charset="uft-8">
            <title>Browserify</title>
        </head>
        <body>
            <button type="button">Alert!</button>
            <script src="bundle.js"></script>
        </body>
        </html>
    
        Листинг 7.2 main.js

        let $=require("jquery")
        $("button").on("click", function() {
            alert("Hello, Browserify!");
        });

Что делать дальше

  1. Запустить Windows PowerShell
  2. Набрать 'browserify main.js --outfile bundle.js'.
  3. Открыть страницу в браузере

Теперь при нажатии кнопки будет выскакивать окно с вашим текстом.

Как разрешить выполнение неподписанного скрипта PowerShell?
В оболочке PowerShell, перед тем как запускать скрипт, выполнить команду, разрешающую выполнение неподписанных скриптов для текущего сеанса оболочки: Set-ExecutionPolicy RemoteSigned -Scope Process.
